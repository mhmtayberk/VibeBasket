# VibeBasket CLI

Apply VibeBasket bundles to your local AI IDEs and CLI tools.

The CLI is built for the last mile:

- apply bundle URLs generated by the VibeBasket web app
- inspect installed MCP, Skill, and Rule surfaces
- search the hosted or self-hosted catalog
- scaffold a local `.vibebasket/` workspace
- roll back from timestamped config backups

## Quick Start

```bash
npx vibebasket apply https://vibebasket.dev/api/bundle/<bundle-id>
```

## Commands

```bash
npx vibebasket apply <url-or-file>
npx vibebasket list
npx vibebasket search postgresql
npx vibebasket doctor
npx vibebasket init
npx vibebasket rollback
```

## How Secrets Work

- Secrets required by MCP servers are resolved locally on your machine.
- The CLI checks `process.env`, `.vibebasket.env`, and OS keychain storage when available.
- If a value is still missing, the CLI prompts you interactively.
- The hosted VibeBasket service does not receive those runtime secrets.

## Self-Hosted Catalogs

By default, `search` uses `https://vibebasket.dev`.

To point the CLI at your own deployment:

```bash
VIBEBASKET_API_URL=https://your-domain.com npx vibebasket search postgresql
```

The same override works for bundle apply flows that use hosted URLs.

## Notes

- Automatic install behavior depends on what each target adapter actually supports.
- Some platforms may not provide native keychain access; the CLI still works and falls back to environment variables and prompts.
