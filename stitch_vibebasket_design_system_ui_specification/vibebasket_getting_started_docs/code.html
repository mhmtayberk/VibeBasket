Getting Started

Welcome to the VibeBasket ecosystem. This guide provides a rapid introduction to initializing your workspace, understanding the core architecture, and deploying your first AI-engineered infrastructure basket.

Installation

To guarantee compatibility with our advanced scaffolding models, ensure you are running an LTS release of Node.js. The primary entry point for VibeBasket is our interactive CLI, which handles dependency resolution automatically.

terminal bash 
npx vibebasket init my-first-basket
# Follow the interactive prompts to configure your environment
info

Global CLI Usage

While npx is recommended for one-off initializations to ensure you use the latest schema definitions, power users may prefer global installation via npm i -g vibebasket for faster repeated access to the vibe alias.

Your first basket

After the initialization sequence successfully scaffolds the directory structure and installs core dependencies, transition into the project root and start the local development daemon. This will bootstrap the interface and establish the local API bindings.

terminal bash 
cd my-first-basket
npm run dev
warning

Network Port Binding

The daemon defaults to binding to port 3000. If this port is actively occupied by another process on your local loopback, the server will autonomously increment the port number (e.g., 3001). Check terminal output for the exact assigned URI.
